Answers

Here we have answers to lifes hardest questions:

1. The Three main diffrences between web1,web2 and web3
  1. web1 consisted of mostly personal pages that were completly static, with web2 websites became more dynamic and things like visit counters and dynamic presentation of data to each user. with web3 this trend grew even more and things like reactive interfaces and live data became more prominent.
  2. in web1 you would only be able to search a specific url while in web2 you can search of a webpage using search engines that index the contents of the web's tags. web3 enables better search results using more complex indexing and searching algprithms that can infer semantic connections and meanings.
  3. starting with web2 it was possible to crawl the entire web - which enabled the emergance of search engines.
2. Medicines side effects Use Case
  1. web1: The user would need to remember a url pointing to a website talking about Medicines side for example - www.med.com/penicillin/side-effects
  2. web2: The user would be able to search for the site through a search engine that indexes web pages acording to the tags they contain thus a search for "penicillin side effects" would lead to web pages containing the search keywords together.
  3. web3: Similarly to web2 the user would search "penicillin side effects" in a web search engine though this time he will be lead to web pages that have a strong semantic connection in thier text to the keywords and their combination.
